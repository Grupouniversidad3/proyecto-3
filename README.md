Adelanto del Proyecto – Directorio de Nómina (RRHH)
Hasta el momento se ha desarrollado la estructura inicial del proyecto siguiendo el enfoque de programación modular. Se creó el menú principal interactivo, el cual permite al usuario seleccionar las diferentes opciones del sistema y permanece en ejecución hasta que se elige la opción de salir.
Como primer avance del CRUD, se implementó la opción de Registrar Empleado. Esta función solicita por consola el nombre del empleado, el cargo que desempeña, el sueldo quincenal y los años de servicio. Con esta información se crea un objeto que representa al empleado y posteriormente se almacena dentro de un arreglo de empleados.
Además, se implementó la persistencia de datos mediante un archivo nomina.json, de forma que cada nuevo registro se guarda automáticamente en el disco y la información no se pierde al cerrar el programa. También se desarrolló un módulo encargado de cargar los datos existentes al iniciar la aplicación y otro para guardarlos después de cada registro.
La aplicación se encuentra organizada en diferentes archivos para separar las responsabilidades de cada módulo, incluyendo el menú principal, la gestión de empleados y el manejo de archivos, facilitando así el mantenimiento y la comprensión del código.
Como siguiente etapa del proyecto, se implementarán las validaciones de las reglas de negocio, las funciones de mostrar, editar y eliminar empleados, así como el módulo de estadísticas solicitado en la evaluación.
